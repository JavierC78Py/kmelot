📚 Guía Completa de Funciones de Ventana (Window Functions) en Oracle
🔍 Query de Ejemplo Explicado
SELECT
    empleado_id,
    departamento_id,
    salario,
    SUM(salario) OVER (
        PARTITION BY departamento_id
        ORDER BY salario
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS acumulado_departamento,
    RANK() OVER (
        PARTITION BY departamento_id
        ORDER BY salario DESC
    ) AS ranking_salarial
FROM empleados;

🧠 Conceptos Fundamentales
¿Qué son las funciones de ventana?
Funciones que realizan cálculos sobre un conjunto de filas relacionadas (ventana) sin agruparlas en una sola fila de resultado.

Estructura Básica

funcion() OVER (
    [PARTITION BY columna]  -- División en grupos
    [ORDER BY columna]      -- Orden dentro del grupo
    [ROWS BETWEEN ...]      -- Definición de la ventana
)

📌 Componentes del OVER()
1. PARTITION BY
Divide los datos en grupos (como GROUP BY pero sin colapsar filas)
-- Ejemplos:
SUM(salario) OVER()                          -- Toda la tabla
SUM(salario) OVER(PARTITION BY depto)         -- Por departamento
SUM(salario) OVER(PARTITION BY depto, cargo)  -- Por depto y cargo

2. ORDER BY
Ordena las filas dentro de cada partición
-- Útil para cálculos acumulativos
SUM(salario) OVER(ORDER BY salario)                    -- Acumulado general
SUM(salario) OVER(PARTITION BY depto ORDER BY salario) -- Acumulado por depto

3. ROWS BETWEEN
Define el rango de filas a incluir en el cálculo
ROWS BETWEEN [inicio] AND [fin]

-- Tipos de límites:
UNBOUNDED PRECEDING   -- Desde el inicio
n PRECEDING           -- n filas antes
CURRENT ROW           -- Fila actual
n FOLLOWING           -- n filas después
UNBOUNDED FOLLOWING   -- Hasta el final

-- Ejemplos:
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW  -- Acumulado hasta ahora
ROWS BETWEEN 3 PRECEDING AND 3 FOLLOWING           -- Ventana de 7 filas
ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING  -- Todas las filas

🎯 Tipos de Funciones de Ventana
1. Funciones de Agregación
SUM()      -- Suma
AVG()      -- Promedio
COUNT()    -- Conteo
MAX()      -- Máximo
MIN()      -- Mínimo

-- Ejemplo:
SELECT 
    salario,
    AVG(salario) OVER(PARTITION BY depto) AS promedio_depto,
    MAX(salario) OVER() AS maximo_global
FROM empleados;

2. Funciones de Ranking
ROW_NUMBER()     -- 1,2,3,4,5 (sin empates, único)
RANK()           -- 1,2,2,4,5 (con empates, salta números)
DENSE_RANK()     -- 1,2,2,3,4 (con empates, sin saltos)
PERCENT_RANK()   -- 0,0.25,0.5,0.75,1 (porcentaje)
NTILE(n)         -- Divide en n grupos

-- Ejemplo:
SELECT 
    salario,
    ROW_NUMBER() OVER(ORDER BY salario) AS fila,
    RANK() OVER(ORDER BY salario) AS ranking,
    DENSE_RANK() OVER(ORDER BY salario) AS ranking_denso,
    NTILE(4) OVER(ORDER BY salario) AS cuartil
FROM empleados;

3. Funciones de Desplazamiento
LAG(columna, n)   -- Valor de n filas antes
LEAD(columna, n)  -- Valor de n filas después
FIRST_VALUE()     -- Primer valor de la ventana
LAST_VALUE()      -- Último valor de la ventana

-- Ejemplo:
SELECT 
    salario,
    LAG(salario, 1) OVER(ORDER BY salario) AS anterior,
    LEAD(salario, 1) OVER(ORDER BY salario) AS siguiente,
    FIRST_VALUE(salario) OVER(ORDER BY salario) AS primero,
    LAST_VALUE(salario) OVER(ORDER BY salario 
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING) AS ultimo
FROM empleados;

💡 Ejemplos Prácticos
1. Comparación con promedio del departamento
sql

SELECT 
    empleado_id,
    departamento_id,
    salario,
    AVG(salario) OVER(PARTITION BY departamento_id) AS promedio_depto,
    salario - AVG(salario) OVER(PARTITION BY departamento_id) AS diferencia
FROM empleados;

2. Top 3 por departamento
WITH ranking AS (
    SELECT 
        empleado_id,
        departamento_id,
        salario,
        ROW_NUMBER() OVER(PARTITION BY departamento_id ORDER BY salario DESC) AS posicion
    FROM empleados
)
SELECT * FROM ranking WHERE posicion <= 3;

3. Porcentaje sobre total
SELECT 
    empleado_id,
    departamento_id,
    salario,
    ROUND(salario * 100.0 / SUM(salario) OVER(), 2) AS porcentaje_global,
    ROUND(salario * 100.0 / SUM(salario) OVER(PARTITION BY departamento_id), 2) AS porcentaje_depto
FROM empleados;

4. Días entre ventas (con fechas)
SELECT 
    fecha_venta,
    monto,
    LAG(fecha_venta) OVER(ORDER BY fecha_venta) AS venta_anterior,
    fecha_venta - LAG(fecha_venta) OVER(ORDER BY fecha_venta) AS dias_diferencia
FROM ventas;

5. Running total por categoría
SELECT 
    categoria,
    producto,
    ventas,
    SUM(ventas) OVER(
        PARTITION BY categoria 
        ORDER BY ventas
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS acumulado_categoria
FROM productos;

⚠️ Errores Comunes y Soluciones
Error	Causa	Solución
ORA-30483	Función de ventana mal ubicada	Verificar sintaxis del OVER()
Resultados incorrectos	Falta PARTITION BY	Especificar partición deseada
Orden inesperado	ORDER BY incorrecto	Revisar criterio de ordenamiento
Ventana equivocada	ROWS BETWEEN mal definido	Verificar límites de la ventana

🚀 Ventajas vs GROUP BY
Característica					Window Functions	GROUP BY
Mantiene filas individuales		✅ Sí				❌ No (colapsa)
Cálculos por grupos				✅ Sí				✅ Sí
Acceso a filas vecinas			✅ Sí				❌ No
Rankings y numeración			✅ Sí				❌ No
Complejidad de sintaxis			Media				Baja

📝 Resumen Rápido
OVER() = 
    [PARTITION BY columna] → Divide en grupos
    [ORDER BY columna]     → Ordena dentro del grupo
    [ROWS BETWEEN ...]     → Define el alcance

Funciones disponibles:
├── Agregación (SUM, AVG, MAX, MIN, COUNT)
├── Ranking (ROW_NUMBER, RANK, DENSE_RANK, NTILE)
└── Desplazamiento (LAG, LEAD, FIRST_VALUE, LAST_VALUE)
